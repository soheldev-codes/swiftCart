১) null বনাম undefined
Undefined: যখন কোনো ভেরিয়েবল ডিক্লেয়ার করা হয় কিন্তু ভ্যালু দেওয়া হয় না। এটা জাভাস্ক্রিপ্টের ডিফল্ট বিহেভিয়ার।

Null: এটা ডেভেলপার নিজে সেট করে। যখন আমরা বোঝাতে চাই যে এই ভেরিয়েবলটা বর্তমানে "খালি" ।

২) map() বনাম forEach()
map(): এটি অ্যারের প্রতিটি উপাদানে কাজ করে এবং শেষে একটি নতুন অ্যারে রিটার্ন করে। (ডাটা ট্রান্সফর্ম করতে লাগে)।

forEach(): এটি শুধু লুপ চালায়, কিন্তু কোনো কিছু রিটার্ন করে না। (শুধু ডাটা প্রিন্ট বা সেভ করতে লাগে)।

৩) == বনাম ===
== এটি শুধু ভ্যালু চেক করে। ডাটা টাইপ আলাদা হলেও সে কনভার্ট করে মেলায়। (যেমন: 5 == "5" is true)।

=== এটি ভ্যালু এবং ডাটা টাইপ দুইটাই চেক করে। কোনো কনভার্ট করে না। (যেমন: 5 === "5" is false)।

৪) async/await এর গুরুত্ব
API থেকে ডাটা আসতে সময় লাগে। জাভাস্ক্রিপ্ট যেন ডাটা আসার আগেই পরের লাইনে চলে না যায়, সেজন্য async/await ব্যবহার করে কোডকে লাইনে দাঁড় করানো হয়। এতে কোড পড়া সহজ হয় এবং এরর হ্যান্ডেল করা সুবিধাজনক।

৫) JavaScript Scope
Global Scope: ভেরিয়েবলটি কোডের যেকোনো জায়গা থেকে পাওয়া যাবে।

Function Scope: ভেরিয়েবলটি শুধু ওই ফাংশনের ভেতরেই কাজ করবে।

Block Scope: let বা const দিয়ে লিখলে সেটি শুধু সেকেন্ড ব্র্যাকেটের { } (যেমন: if বা for loop) এর ভেতরেই সীমাবদ্ধ থাকবে।
